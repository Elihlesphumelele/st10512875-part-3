This CSS stylesheet was designed to create a modern, professional, and fully responsive website for a security services company. The stylesheet begins with a reset section that removes the default margins and padding applied by web browsers and sets the box-sizing property to border-box. This ensures that all elements are sized consistently and makes it easier to control layouts throughout the website. 

The body styling establishes the overall appearance of the website by setting the font family to Arial, Helvetica, and sans-serif, which provides a clean and professional look. It also applies a light grey background colour, dark text colour, and comfortable line spacing to improve readability and user experience. 

The header section creates a dark blue banner at the top of the website that contains the company branding and navigation menu. The logo container uses Flexbox to align the company logo, company name, and tagline neatly. The logo is given fixed dimensions and object-fit properties to maintain its appearance regardless of image size. The company name is styled with a large font size, while the tagline uses a lighter blue colour to create visual contrast. 

The navigation bar provides links to the different pages of the website. Flexbox is used to arrange the navigation links horizontally with equal spacing. The links are displayed in white with bold text, and a smooth transition effect changes their colour to bright blue when the user hovers over them or when a page is active. This improves navigation and provides visual feedback to users. 

A hero section appears at the top of each page to capture visitors' attention. It uses a large background image related to security services with a dark transparent gradient overlay. The overlay improves text visibility while maintaining the visual impact of the image. The hero section contains large headings and descriptive text that introduce the page content. Flexbox is used to centre the content both vertically and horizontally, creating a professional and balanced layout. 

The stylesheet includes container classes that control the width of page content. These containers limit content to a maximum width while keeping it centred on the screen. This prevents text and images from stretching too far across large displays and improves readability. 

Several card-based layouts are used throughout the website. Information cards, service cards, team cards, news cards, and testimonial cards all share similar design principles. Each card uses a white background, rounded corners, padding, and subtle box shadows to create depth and separation from the page background. Hover effects lift the cards slightly using CSS transforms, making the website more interactive and engaging. 

The team section is designed to introduce company staff members. Team cards display employee photographs, names, and positions. Images are resized using object-fit properties to maintain quality and consistency. Hover animations provide a modern user experience and draw attention to individual team members. 

The services section uses a grid layout to display the company's security services. CSS Grid automatically adjusts the number of columns based on screen size, ensuring that service cards remain properly aligned on both large and small screens. Each service card contains an icon, service title, and description, helping visitors quickly understand the company's offerings. 

The contact page includes several specialised components. Contact information is displayed in information cards that present phone numbers, email addresses, and physical locations. A contact form allows users to submit enquiries by entering their details and messages. Input fields and text areas are styled with borders, padding, and rounded corners to create a clean appearance. A submit button uses colour transitions to provide visual feedback when hovered over. The page also includes a map section containing an embedded map displayed inside a styled container. 

The news section is designed to present company updates, announcements, and industry information. News cards include large featured images, publication dates, headlines, and article summaries. The date labels are styled as coloured badges to make publication dates easy to identify. Hover animations make the news cards feel interactive and encourage user engagement. 

The testimonials section displays feedback from clients. Testimonial cards contain quotation symbols, customer comments, and client names. A feature section highlights the company's strengths using feature boxes arranged in a responsive grid. Hover effects change the background colour of feature boxes to improve user interaction and draw attention to important information. 

The footer section appears at the bottom of every page and provides additional company information, contact details, and copyright information. It uses the same dark blue colour scheme as the header to maintain visual consistency throughout the website. 

A significant feature of the stylesheet is its responsive design implementation. Media queries are used to adjust layouts when the screen width falls below 768 pixels. On smaller devices, Flexbox layouts change from horizontal to vertical arrangements, navigation menus stack vertically, images resize appropriately, and text sizes are reduced to improve readability. Form sections, cards, and content containers also adjust their widths to fit smaller screens. These responsive design techniques ensure that the website remains accessible, functional, and visually appealing on desktops, tablets, and smartphones. 

Overall, this CSS stylesheet creates a professional, modern, and user-friendly security company website by combining consistent branding, responsive layouts, interactive hover effects, organised content sections, and visually appealing design elements. The use of Flexbox, CSS Grid, media queries, transitions, shadows, and background overlays helps deliver an attractive user experience while ensuring compatibility across different devices and screen sizes.

This JavaScript file for APEX PROTECT is responsible for improving user experience through three main features: automatic navigation highlighting, detailed contact form validation, and toast-style notifications that give feedback to the user. It runs once the page has fully loaded using a DOMContentLoaded event listener, which ensures all HTML elements are available before the script tries to interact with them. 

The first feature automatically highlights the active navigation link. The script determines the current page by reading the URL using window.location.pathname and extracting the filename (for example, index.html or contact.html). It then loops through all navigation links inside the .navbar using querySelectorAll('.navbar a'). For each link, it removes any existing ‚Äúactive‚Äù class to reset the state. It then compares the link‚Äôs href value (also reduced to just the filename) with the current page. When a match is found, the script adds the active class to that link. This makes the navigation menu automatically reflect the page the user is currently viewing, improving usability and eliminating the need to manually set active states in each HTML file. 

The second and most complex feature is the contact form validation system. The script first selects the form using .contact-form. If the form does not exist on the page, it stops running (return), which prevents errors on pages that do not contain the form. It then defines three helper functions to manage form states. The setError() function marks a field as invalid by adding an input-error class and removing input-success. It also creates or updates a small <span> element under the input to display a specific error message such as ‚ÄúEmail is required‚Äù or ‚ÄúMessage must be at least 10 characters.‚Äù The setSuccess() function does the opposite: it removes error styling, adds a success class, and hides any error message. The clearState() function resets the field completely by removing both success and error styles when the user starts typing again. 

After setting up helpers, the script defines three validation functions: validateName, validateEmail, and validateMessage. The name validation ensures the field is not empty, is at least 2 characters long, and contains only valid characters (letters, spaces, hyphens, and apostrophes) using a regular expression. The email validation checks that the field is not empty and matches a standard email pattern using a regex like /^[^\s@]+@[^\s@]+\.[^\s@]+$/. The message validation ensures the user has typed something meaningful by requiring at least 10 characters. Each function either calls setError() with a custom message or setSuccess() if the input is valid, and returns true or false accordingly. 

The script then connects these validations to user interactions. It adds blur event listeners so that when the user leaves a field, it is immediately validated. This provides instant feedback without waiting for form submission. It also adds input event listeners so that whenever the user starts typing again, any previous error or success styling is cleared, allowing the field to reset visually in real time. 

When the user submits the form, the script prevents the default submission using e.preventDefault() so the page does not reload. It then runs all three validation functions. If all fields are valid, the script simulates a successful submission: it shows a success toast message, resets the form using form.reset(), and clears all validation styles from the fields. If any field is invalid, it shows an error toast and automatically moves the cursor to the first field with an error using querySelector('.input-error'), helping the user quickly fix the issue. 

The third feature is the toast notification system, which provides temporary popup messages. The showToast() function first checks if a toast element already exists on the page (apex-toast). If not, it creates one dynamically and adds it to the document body. It then updates the message text and applies styling based on the type of message (success or error). The toast is shown by adding a show class, and a timer (setTimeout) automatically hides it after 4 seconds. If a new toast is triggered before the previous one disappears, the previous timer is cleared to prevent conflicts. 

Overall, this script significantly enhances the website by making navigation more intuitive, enforcing strong and user-friendly form validation, and improving feedback through modern notification messages.
